# Память агента team

Это персистентное хранилище командного ассистента. Каждая папка — отдельный
слой памяти с определённым назначением.

## Структура

```
memory/
├── profile.md              # краткое резюме о текущем состоянии команды
├── index.md                # навигация по wiki
├── log.md                  # append-only журнал крупных событий
│
├── wiki/                   # структурированные знания (обновляет Dream)
│   ├── entities/          # страницы на каждого участника (имя, роль, история)
│   ├── concepts/          # задачи, решения, проекты, процессы
│   │   ├── tasks.md       # активные задачи команды
│   │   ├── decisions.md   # принятые решения с контекстом
│   │   ├── promises.md    # трекер обещаний (план/факт)
│   │   └── company/       # база знаний о компании (наполняется вручную)
│   └── synthesis/         # выводы и кросс-референсы
│       └── daily/         # ежедневные саммари чата
│
├── groups/                 # per-chat runtime config (создаётся онбордингом)
│   └── {chat_id}/
│       ├── settings.json  # allowed_topic, таймзона, расписание, язык
│       └── daily/         # сырой лог всех сообщений чата за день
│
├── reminders/              # динамические напоминания от участников
│   └── {chat_id}.json     # массив напоминаний с ISO timestamps
│
├── raw/                    # сырьё для Dream (conversations.jsonl и др.)
├── daily/                  # персональные daily notes (DM режим)
├── sessions/               # session ID для --resume
├── outbox/                 # файлы обратно текущему пользователю (sync-ответ)
├── dispatch/               # фоновые сообщения с явным chat_id+thread_id
├── delegation/             # запросы к другим агентам через FleetBus
└── stats/                  # метрики, использование токенов
```

## Что обновляет человек, а что — агент

**Человек (через Telegram-онбординг или вручную):**
- `wiki/concepts/company/*.md` — знания о компании
- `groups/{chat_id}/settings.json` — создаётся онбордингом через master-агента
- `profile.md` — можно править, но агент тоже его обновляет

**Агент (автоматически):**
- `wiki/entities/`, `wiki/concepts/tasks.md`, `wiki/concepts/decisions.md`,
  `wiki/concepts/promises.md` — обновляет Dream и по ходу чата
- `wiki/synthesis/daily/*.md` — ночное саммари
- `reminders/{chat_id}.json` — пишет скилл reminders
- `groups/{chat_id}/daily/*.md` — тихое логирование всех сообщений чата
- `dispatch/*.json` — фоновые сообщения в Telegram, которые `dispatcher.py` заберёт и опубликует
- `index.md` — обновляется при любых изменениях wiki

## Git

Эта папка — отдельный git-репо (`.git` внутри). Все изменения коммитятся
автоматически после Dream-прохода или выполнения cron-задачи. История
сохраняется, откат возможен через `git log` / `git restore`.

## Как сюда попадает этот README

Файл приехал из `agents/team/memory_template/README.md` при первом запуске
агента через `memory.ensure_dirs()` → `_seed_from_template()`. Шаблон —
часть публичного репозитория my-claude-bot, а живые данные — нет.
Если что-то тут поправишь, твои изменения сохранятся: seeding никогда
не перезаписывает существующие файлы.
