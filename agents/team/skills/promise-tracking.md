# Skill: Promise Tracking

## Принцип

Обещание — это любое добровольное обязательство участника чата сделать что-то к какому-то моменту. Скилл отслеживает жизненный цикл обещания: **взято → в работе → закрыто** (или **просрочено → перенесено**).

## Парсинг обещания

Паттерны, которые считаются обещанием:

- «Я возьму X»
- «Я сделаю X к Y»
- «Обещаю X до Y»
- «Я беру это на себя»
- «Я делаю X сегодня/завтра/до пятницы»
- Ответ на утренний стендап с планом на день

**Что извлекать:**

| Поле | Источник |
|------|---------|
| `who` | tg username автора сообщения |
| `what` | краткое описание задачи (1 предложение) |
| `due` | дедлайн в ISO (если сказано «к Y» — распарсить, иначе конец дня из `settings.json` → `daily_sync.day_end_time`) |
| `source_message_id` | id исходного сообщения (если доступно в контексте) |
| `created_at` | ISO timestamp сейчас |

## Запись обещания

1. Прочитай `wiki/concepts/promises.md`.
2. Найди секцию `## YYYY-MM-DD` для сегодня. Если нет — создай.
3. Добавь новый пункт в формате, описанном в промисах:
   ```
   - [ ] **@username** — описание задачи — дедлайн: YYYY-MM-DD HH:MM — добавлено: ISO
         └ источник: message_id или краткая цитата
   ```
4. Коротко подтверди в чат: «📝 Записал: @user — задача — до HH:MM».

## Отметка выполнения

Паттерны:
- «Сделал X» / «Готово» / «Закрыл X» / «Done»
- Явное упоминание задачи из `promises.md`

1. Прочитай `promises.md`, найди соответствующий `[ ]`.
2. Замени на `[x]`.
3. Добавь строку `└ закрыто: ISO timestamp — <комментарий из сообщения>`.
4. Подтверди: «✅ Закрыл: <задача>».

Если по сообщению нельзя однозначно понять, какая именно задача закрывается:
- Переспроси («какую задачу закрыл? вижу несколько активных у тебя»).
- Не гадай.

## Перенос невыполненного

Вызывается из `daily-sync` вечером:

1. Для каждого `[ ]` в сегодняшней секции, где `due < now()` и нет `закрыто:`:
   - Добавь строку `└ перенесено на YYYY-MM-DD — причина: не отмечено как выполненное вечером`.
   - Оставь исходный `[ ]` как есть (для истории).
   - Добавь новую запись в секцию `## завтра` с тем же описанием и новым дедлайном.

## Запросы статуса

Паттерны:
- «Что у меня?» / «Что я обещал?» / «Мои задачи»
- «Что у @user?» (если `settings.json` → `privacy.hide_from_others: false`)
- «Что просрочено?» / «Что вчера не закрыли?»

Ответ: короткий список с датами и статусами. Детали — если спросят.

## Ограничения

- **Не записывай одно и то же дважды.** Перед добавлением проверь, нет ли уже такого обещания от этого участника сегодня.
- **Не придумывай дедлайны.** Если не сказано — ставь конец рабочего дня (`day_end_time`).
- **Уважай приватность.** По умолчанию участник видит свои обещания и общий план команды. Чужие детали — только если не запрещено в `settings.json`.
- **Обещание — это не задача.** Задача живёт в `wiki/concepts/tasks.md`, обещание — в `promises.md`. Одна задача может породить несколько обещаний.
