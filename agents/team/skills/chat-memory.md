# Skill: Chat Memory

## Принцип

Бот **читает всё**, что происходит в разрешённом топике группы, но **отвечает только на упоминания и reply**. Всё прочитанное становится топливом для Dream-пайплайна и базой для Q&A.

Этот скилл описывает, как team работает с непрерывным потоком сообщений.

## Что сохраняется автоматически (без участия LLM)

Telegram-мост на стороне Python уже:
1. Логирует каждое сообщение в `groups/{chat_id}/daily/YYYY-MM-DD.md` (тихое логирование).
2. Кладёт сырой JSONL в `raw/conversations/` для последующей обработки Dream.

LLM вызывается только когда:
- Участник упомянул бота или ответил на его сообщение → обычный цикл ответа.
- Сработал Dream (каждые 3 часа) → два прохода над новыми сообщениями.
- Сработал cron `schedule_check` → проверка стендапов/напоминаний.

## Что делает этот скилл при Dream-проходе

Dream Phase 1 (haiku) уже извлекает факты из raw-сообщений по шаблону `templates/dream_phase1.md`. Этот скилл задаёт **что** извлекать именно в групповом режиме:

1. **Новые участники** → страница в `wiki/entities/<username>.md` (имя, tg username, роль если упомянута, когда появился).
2. **Роли и зоны ответственности** → обновить существующую entity-страницу.
3. **Задачи и их владельцы** → `wiki/concepts/tasks.md` + обновление `promises.md`.
4. **Принятые решения** → `wiki/concepts/decisions.md` (дата, автор, контекст).
5. **Темы обсуждений** → если тема появилась впервые, создать `wiki/concepts/<topic>.md`; если уже была — обновить.
6. **Блокеры и проблемы** → `wiki/concepts/blockers.md` или внутри нужной темы.

## Перекрёстные ссылки

После каждого Dream-прохода Phase 2 (sonnet) должен:

1. Проверить, упоминается ли новая сущность в старых страницах → добавить wiki-ссылки в обе стороны.
2. Обновить `wiki/synthesis/` — выводы вида «Задача X связана с решением Y».
3. Обновить `index.md` — актуальную навигацию по всем страницам.

## Ночное саммари

Cron-задача `schedule_check` раз в сутки (время берётся из `settings.json` → `daily_sync.nightly_summary_time`, дефолт 23:30 локального) запускает:

1. Прочитать `groups/{chat_id}/daily/YYYY-MM-DD.md`.
2. Прочитать все обновления wiki за сегодня (через `git log` в `memory/`).
3. Сформировать саммари дня в `wiki/synthesis/daily/YYYY-MM-DD.md`:
   - Основные темы
   - Принятые решения
   - Новые задачи
   - Закрытые задачи
   - Ссылки на обновлённые wiki-страницы
4. Обновить `index.md` → секцию «Последние саммари».

Саммари не отправляется в чат (это внутренний артефакт для памяти).

## Q&A на основе памяти

Когда участник упоминает бота и задаёт вопрос типа «что мы решили по X?», «кто отвечает за Y?», «когда дедлайн Z?»:

1. Сначала — `Grep` по `wiki/` для точного матча по ключевым словам.
2. Если не найдено — `Read` `index.md` и попытка найти релевантный концепт.
3. Если не найдено — честно скажи «не помню, первый раз слышу» и предложи записать.
4. Если найдено — ответь коротко + дай ссылку на wiki-страницу.

## Ограничения

- **Не вмешивайся без упоминания.** Даже если видишь ошибку в чате — молчи, если тебя не спросили.
- **Не раскрывай приватное.** Если в `settings.json` → `privacy.hide_from_others: true` — не отвечай участнику на вопросы о других, только о нём самом.
- **Не выдумывай.** Если нет данных — скажи «не знаю».
