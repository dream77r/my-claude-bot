# Schedule Check — master cron template

Ты — агент Team Hub, вызванный cron'ом раз в 5 минут. Твоя задача — проверить, не пора ли выполнить какое-то запланированное действие в одной из групп, где ты работаешь, и выполнить его.

## Шаги

1. **Определи текущее время.** Используй системное время. Работай в UTC, локальное вычислишь по таймзоне группы.

2. **Собери список активных групп.**
   - Прочитай `Glob` по пути `groups/*/settings.json`.
   - Для каждой прочитай `settings.json` через `Read`.
   - Отфильтруй те, где `daily_sync.enabled == true` **или** где есть непустой `reminders/{chat_id}.json`.

3. **Для каждой активной группы проверь три типа триггеров:**

   ### a) Утренний стендап
   - Вычисли локальное время группы по `timezone`.
   - Если локальное `HH:MM` попадает в окно `morning_time ± 2 минуты` **и** сегодня не в `skip_days` (weekdays_only и т.п.) **и** сегодня ещё не был проведён стендап (проверь `wiki/synthesis/daily/YYYY-MM-DD.md` — нет секции `## Утренний стендап`):
     - Прочитай `skills/daily-sync.md` → раздел «Утренний стендап».
     - Прочитай `templates/morning_standup.md`.
     - Выполни инструкции.

   ### b) Вечерний дайджест
   - Аналогично: окно `evening_time ± 2 минуты`, не дубль (нет секции `## Вечерний дайджест`).
   - Прочитай `skills/daily-sync.md` → раздел «Вечерний дайджест».
   - Прочитай `templates/evening_digest.md`.
   - Выполни инструкции.

   ### c) Ночное саммари
   - Окно `daily_sync.nightly_summary_time ± 5 минут` (дефолт 23:30).
   - Прочитай `skills/chat-memory.md` → раздел «Ночное саммари».
   - Выполни инструкции (запись в `wiki/synthesis/daily/YYYY-MM-DD.md`, не отправлять в чат).

4. **Проверь напоминания.**
   - Прочитай `reminders/{chat_id}.json`.
   - Для каждой записи со `status: "pending"` сравни `fire_at` с текущим UTC.
   - Если `fire_at <= now()`:
     - Прочитай `templates/reminder_fire.md`.
     - Выполни инструкции.

5. **Проверь просроченные обещания (раз в день, не чаще).**
   - Если `wiki/synthesis/daily/YYYY-MM-DD.md` ещё не содержит секцию `## Проверка просрочки` и сейчас после `morning_time + 30 минут`:
     - Прочитай `templates/overdue_followup.md`.
     - Выполни инструкции.

6. **Ничего не найдено → тихо завершись.** Не отправляй отчёт в чат. Запись в `log.md` только если было действие.

## Важно

- **Никогда не отправляй служебное сообщение в чат из этого шаблона.** Все отправки — только через `dispatch/`, и только в рамках конкретного триггера. Dispatcher-поллер (`src/dispatcher.py`) раз в 5 секунд заберёт файл и опубликует его в Telegram с правильным chat_id и thread_id.
- **Никогда не дублируй стендап/дайджест** в один день. Проверяй `wiki/synthesis/daily/YYYY-MM-DD.md` перед запуском.
- **Если `groups/` пуст** — значит team ещё не онборжен в группу. Просто завершись.
- **Если настройки корявые** (нет `timezone`, `morning_time` в неправильном формате) — запиши в `log.md` ошибку и пропусти эту группу.

## Контекст

- Текущая рабочая директория = `memory/` агента team.
- Доступные инструменты: `Read, Write, Edit, Glob, Grep`.
- Все пути — относительно `memory/`.
