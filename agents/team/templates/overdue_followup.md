# Overdue Followup Template

Напоминание о вчерашних незакрытых задачах. Запускается утром после стендапа.

## Когда запускать

Из `schedule_check.md` раз в день, через ~30 минут после `morning_time`, если:
- Утренний стендап уже прошёл сегодня (секция есть).
- Секция `## Проверка просрочки` ещё не записана в `wiki/synthesis/daily/YYYY-MM-DD.md`.
- Во вчерашнем `promises.md` есть незакрытые `[ ]` без строки `перенесено на:` сегодняшнюю дату.

## Что прочитать

1. `groups/{chat_id}/settings.json`.
2. `wiki/concepts/promises.md` — секции за вчера и за сегодня.

## Логика

1. Собери все вчерашние `[ ]`, которые были перенесены на сегодня (см. `└ перенесено на <сегодня>` в строках вчерашней секции).
2. Проверь, видел ли ты реакцию участника на них в сегодняшнем утреннем стендапе (скан `groups/{chat_id}/daily/<сегодня>.md` на упоминания).
3. Если участник не отреагировал — вставь его в список «тихие».

## Шаблон сообщения

### Русский

```
⏰ Напоминание о вчерашних задачах

{для каждого "тихого" участника: "@user — <задача>"}

Подскажите, что с этим?
```

### English

```
⏰ Yesterday's carry-overs

{for each silent member: "@user — <task>"}

Any update on this?
```

## Действия

1. Если «тихих» нет → не отправляй, только запиши в `wiki/synthesis/daily/YYYY-MM-DD.md` секцию `## Проверка просрочки` с нулевым результатом.
2. Если есть → запиши в `dispatch/{chat_id}_{thread_id}_{iso}.json` с `source: "overdue_followup"`.
3. Всегда создавай секцию `## Проверка просрочки` в саммари дня — это маркер, что followup для сегодня уже прошёл.

## Ограничения

- Максимум один followup в день.
- Не пинать одного и того же человека больше 3 дней подряд за одну и ту же задачу. На 4-й день — эскалация: тегнуть `admins` из `settings.json` с пометкой «задача висит N дней».
- Если `daily_sync.overdue_followup_enabled: false` в settings — вообще пропустить.
