# Meta-шаблон карточки сущности

> Это шаблон для создания **шаблонов** конкретных сущностей. Используется скиллом
> `vault-init` для генерации `templates/{entity_type}.md` под конкретный домен.
> Не копируется напрямую в vault — это референс.

## Структура карточки любой сущности

```markdown
---
title: "{{HUMAN_READABLE_NAME}}"
description: "{{SHORT_DESCRIPTION}}"
created: {{DATE_CREATED}}
updated: {{DATE_UPDATED}}
type: {{ENTITY_TYPE}}           # из entities в .vault-config.json
tags: [{{TAGS_FROM_TAXONOMY}}]
status: active | archived | draft
---

# {{HUMAN_READABLE_NAME}}

## Основная информация

<!-- Поля, уникально идентифицирующие сущность. Для организаций — ИНН/КПП/ОГРН.
     Для персон — имя/должность/контакты. Для проектов — название/сроки/ответственный.
     Список обязательных полей определён в .vault-config.json → required_fields_by_type -->

- **{{FIELD_1}}:** {{VALUE_1}}
- **{{FIELD_2}}:** {{VALUE_2}}

## Роль / Контекст

<!-- Чем эта сущность является в твоём домене. Для контрагента: поставщик или
     покупатель, чего именно. Для проекта: цель, статус. Для персоны: должность,
     отношение к делу. -->

## Связи

<!-- Минимум одна исходящая wiki-ссылка на другую сущность или документ.
     Чем богаче сеть связей — тем полезнее архив при поиске. -->

- [[{{RELATED_SLUG_1}}|{{RELATED_DISPLAY_NAME_1}}]] — {{RELATION_DESCRIPTION}}
- [[{{RELATED_SLUG_2}}]]

## Документы

<!-- Ссылки на все карточки документов, в которых упомянута эта сущность.
     Обновляется автоматически при ingest каждого нового документа. -->

- [[{{DOC_SLUG_1}}]] — {{DOC_DESCRIPTION}}

## Заметки

<!-- Свободная форма: наблюдения, контекст, предупреждения, история изменений
     условий. Здесь хранится "мягкое знание", которое не влезает в структурные
     поля. -->
```

## Правила для vault-init при генерации шаблона под конкретную сущность

1. Замени `{{ENTITY_TYPE}}` на конкретный тип (например, `counterparty`).
2. Определи минимальный список обязательных полей в секции "Основная информация"
   и запиши их в `.vault-config.json → required_fields_by_type[{entity_type}]`.
3. Переименуй секцию "Роль / Контекст" под домен (например, "Роль в договорах"
   для контрагента, "Цель проекта" для проекта).
4. Добавь доменные секции, если они типичны (например, "Банковские реквизиты"
   для юрлица).
5. Сохрани комментарии `<!-- ... -->` как подсказки для LLM при заполнении.
