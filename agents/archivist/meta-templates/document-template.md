# Meta-шаблон карточки документа

> Это шаблон для создания шаблонов карточек документов под конкретный подтип
> (договор, акт, счёт, накладная и т.д.). Используется скиллом `vault-init`
> и `schema-evolve` для генерации `templates/{document_subtype}.md`.

## Структура карточки документа

```markdown
---
title: "{{DOC_TYPE}} №{{DOC_NUMBER}} от {{DOC_DATE}}"
description: "{{SHORT_DESCRIPTION}}"
created: {{DATE_CREATED}}
updated: {{DATE_UPDATED}}
type: document
document_subtype: {{SUBTYPE}}       # contract | act | invoice | waybill | ...
tags: [{{TAGS_FROM_TAXONOMY}}]
status: active | expired | expiring-soon | draft | cancelled
source_file: "raw/{{DATE}}_{{SLUG}}.pdf"
doc_date: {{YYYY-MM-DD}}             # дата подписания/выставления
entities:
  {{ENTITY_TYPE_1}}: [[{{SLUG_1}}]]
  {{ENTITY_TYPE_2}}: [[{{SLUG_2}}]]
---
```

## Правила для vault-init / schema-evolve

1. `document_subtype` — это значение из `document_types` в `.vault-config.json`.
2. Набор секций зависит от подтипа:
   - Договор: Основные данные, Стороны, Предмет, **Условия оплаты**,
     **Штрафные санкции**, Расторжение, Связанные документы, Заметки
   - Акт сверки: Основные данные, Стороны, **Период**, **Итоговая сумма**,
     Подписи, Заметки
   - Счёт: Основные данные, Стороны, Предмет, **Реквизиты оплаты**, Заметки
   - Накладная: Основные данные, Стороны, **Позиции** (таблица), Заметки
3. Обязательные поля для каждого подтипа — в `.vault-config.json →
   required_fields_by_type[{document_subtype}]`.
4. Любой документ обязан быть связан хотя бы с одной сущностью через `entities`.
