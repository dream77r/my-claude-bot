# Архив (vault)

Это точка входа в твой архив документов. Сейчас он пуст — ждёт инициализации.

## Первый шаг — инициализация

Vault ещё не настроен под твой домен. Напиши Архивариусу в Telegram:

> Привет, давай настроим архив

Он проведёт тебя через короткий диалог (5-10 минут) и создаст структуру под
твои задачи. Три режима инициализации:

1. **Импорт существующего архива** — кинь ему zip или папку с документами, он
   проанализирует структуру и предложит схему
2. **С нуля через вопросы** — он спросит про домен, сущности, типы документов
3. **От примера** — возьми один из готовых шаблонов в `examples/` (например,
   `small-business` для малого бизнеса с несколькими юрлицами)

## После инициализации

Здесь появится:
- Список сущностей твоего домена (контрагенты, проекты, персоны — зависит от
  настройки)
- Количество карточек по каждому типу
- Последние добавленные документы
- Ближайшие даты истечения (если применимо)
- Статус здоровья архива

## Что делать дальше

После инициализации ты сможешь:
- **Загружать документы** — просто кидай файлы Архивариусу в чат
- **Искать** — задавай вопросы ("ИНН контрагента X", "все договоры за март")
- **Получать напоминания** — о приближающихся дедлайнах
- **Развивать схему** — добавлять новые сущности, типы, теги по мере роста архива

См. также:
- `CLAUDE.md` — инструкции, которыми руководствуется Архивариус
- `SCHEMA.md` — доменная схема (появится после vault-init)
- `log.md` — журнал всех действий в архиве
