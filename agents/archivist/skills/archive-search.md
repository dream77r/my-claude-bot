# Skill: archive-search — поиск по архиву

## Когда активировать

Любой вопрос пользователя, который требует чтения данных из vault'а:
- Точечный факт: "ИНН X", "телефон Y", "контактное лицо Z"
- Срез: "все просроченные договоры", "все документы за март с контрагентом X"
- Аналитика: "сколько договоров с X?", "у каких наших ООО есть договоры с X?"
- Поиск: "есть ли у нас документ X?"

## Предусловия

1. Vault инициализирован (`.vault-config.json → initialized: true`)
2. Пользователь имеет минимум уровень Reader (проверка по Telegram ID)

## Классификация запроса

Прежде чем искать, определи тип запроса — это определяет формат ответа.

### Тип 1 — Lookup (точечный факт)

**Признаки:** запрос содержит конкретную сущность и конкретное поле.
- "ИНН {контрагент}"
- "телефон {контрагент}"
- "адрес {юрлицо}"
- "когда истекает договор {номер}"
- "какая сумма по договору с {контрагент}"

**Как искать:**
1. Определи сущность (или документ) по упомянутому в запросе названию/номеру
2. Найди её карточку через Glob + Grep по frontmatter/тела
3. Если несколько кандидатов (двусмысленность) — уточни у пользователя
4. Прочитай карточку, извлеки запрошенное поле

**Формат ответа:**
```
{значение поля}
{короткий контекст — откуда: "из карточки контрагента X" или "из договора №N от YYYY-MM-DD"}
```

Пример:
```
ИНН: 5504104520
Из карточки контрагента ООО "Пример"
```

### Тип 2 — Filter (срез)

**Признаки:** запрос про "все X" или "X с условием Y".
- "все просроченные договоры"
- "все договоры с {контрагент}"
- "акты сверки за март"
- "документы по {проект}"

**Как искать:**
1. Определи какой тип искать (`contract`, `document`, подтип)
2. Определи условия фильтра (статус, дата, связанная сущность, тег)
3. Прочитай `_index.md` соответствующей папки — часто этого достаточно
4. Если нет — Glob по папке, Read каждой карточки, фильтр по frontmatter
5. Собери результаты

**Формат ответа:**
```
Нашёл {N} {тип} по условию:

- {title или номер+дата} — {краткое описание} — {статус}
- ...

{Если больше 10 — показать первые 10 и сказать "и ещё X"}
{Если ничего не найдено — "Ничего не нашёл по этому условию"}
```

Пример:
```
Нашёл 3 договора с ООО "Пример":

- №876-03/2026 от 10.03.2026 — бухгалтерское обслуживание — активен
- №877-03/2026 от 10.03.2026 — ЭДО — активен
- №881-05/2026 от 15.05.2026 — юр. услуги — истекает через 12 дней
```

### Тип 3 — Traversal (графовая аналитика)

**Признаки:** вопрос требует подсчёта, пересечений, или обхода связей.
- "сколько договоров с X?"
- "у каких наших ООО есть договоры с X?"
- "какие точки обслуживает X?"
- "все связи контрагента X"

**Как искать:**
1. Найди начальную сущность (карточку X)
2. Обойди её wiki-ссылки (секции "Договоры", "Документы", "Связи")
3. Для каждой связи — прочитай связанную карточку, собери нужные поля
4. Сагрегируй (посчитай, сгруппируй, покажи связи)

**Формат ответа:**
```
{агрегированное число или список}

{подробности — с какими сущностями связано, какие условия}
```

Пример:
```
У ООО "Пример" 4 договора с нашими юрлицами:

- Наше ООО "Alpha" — 2 договора (бухгалтерия, ЭДО)
- Наше ООО "Beta" — 2 договора (бухгалтерия, ЭДО)

Все договоры — обслуживание (бухгалтерия и ЭДО). Самый поздний — от 13.03.2026.
```

## Специальный случай — запрос "покажи/пришли сам файл"

**Триггеры:** "пришли файл", "дай сам документ", "покажи PDF", "скачай"

**Действие:** прикрепить файл из `raw/` напрямую в ответ через Telegram.
Это единственный случай, когда пользователь получает сырой файл.

**Формат:** прикрепить файл + короткое подтверждение ("Прикрепляю договор №X
от YYYY-MM-DD").

## Специальный случай — запрос "откуда?"

**Триггеры:** "откуда?", "из какого документа?", "источник?"

**Действие:** назвать документ, из которого была взята предыдущая информация,
в человекочитаемом виде. НЕ технический путь.

Пример:
```
Из договора №876-03/2026 от 10.03.2026 (бухгалтерия, ООО "Пример" — ООО "Alpha").
Если нужен сам файл — скажи "пришли".
```

## Общие правила поиска

1. **Начинай с `_index.md`.** Индексы — это сводные таблицы, они дают быстрый
   ответ на большинство filter-запросов без чтения каждой карточки.
2. **Frontmatter важнее тела.** Для lookup'а точечных полей сначала проверь
   frontmatter, потом тело.
3. **Wiki-ссылки — это граф.** Для traversal-запросов иди по ссылкам, не
   сканируй весь vault.
4. **Если ничего не нашёл — скажи честно.** Не выдумывай. "Не нашёл у себя X"
   — это нормальный ответ.
5. **При двусмысленности — спроси.** Если "контрагент ООО Пример" — а их два с
   разными ИНН, переспроси: "у тебя два ООО Пример: {различающее поле 1} и
   {поле 2}. Какой?"

## Формат ответов (общие правила)

- **Средняя длина**, не односложный, но без лирики
- **Никаких технических путей** (`wiki/...md`, `raw/...pdf`)
- **Человеческие ссылки на источники** ("из договора №X от YYYY-MM-DD")
- В групповом чате — максимально компактно (5 строк или меньше)
- В личном чате — можно развернуть, если это помогает

## Запреты

- НЕ показывай содержимое vault'а пользователю уровня Reader, если это
  противоречит политике доступа (в будущем — `access.readers_scope`)
- НЕ выдумывай поля, которых нет в карточках
- НЕ прикрепляй файлы без явного запроса
- НЕ показывай пути к файлам в ответах
- При противоречиях в данных — показывай обе версии, не выбирай молча
