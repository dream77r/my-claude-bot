# Пример — Малый бизнес

Готовый шаблон доменной схемы для малого бизнеса с несколькими юрлицами и
физическими точками (кафе, магазины, офисы, склады).

## Когда использовать

Этот пример подходит, если у тебя:
- Одно или несколько собственных ООО/ИП
- Одна или несколько физических точек (возможно, разные точки у разных ООО)
- Работа с контрагентами (поставщики, арендодатели, сервисные компании)
- Нужно отслеживать договоры, первичку, сроки
- Условия договоров могут различаться между точками или юрлицами

## Что входит в пример

- **SCHEMA.md** — полная доменная схема с 5 сущностями
- **vault-config.json** — таксономия, обязательные поля, правила дедупликации
- **templates/** — шаблоны карточек для каждой сущности:
  - `legal-entity.md` — наши юрлица
  - `outlet.md` — физические точки
  - `counterparty.md` — контрагенты
  - `contract.md` — договоры
  - `document.md` — первичка (акты, счета, накладные, платёжки)

## Сущности

```
legal_entity (наши ООО)
    │
    └── owns ──▶ outlet (физическая точка)
                    │
                    └── has contracts ──▶ contract ──▶ counterparty
                                              │
                                              └── generates ──▶ document (акты, счета)
```

## Таксономия тегов (коротко)

- **Отношение:** supplier / customer / rental / service
- **Категория:** food / logistics / real-estate / utilities / accounting-legal / equipment
- **Оплата:** prepaid / postpaid / mixed / auto-renewal
- **Автотеги:** `le-{slug}` (по юрлицу), `outlet-{slug}` (по точке)

## Как взять за основу

При `vault-init` выбери режим "от примера" и укажи `small-business`. Архивариус
скопирует схему, шаблоны и таксономию в твой `memory/`, а затем проведёт тебя
через настройку под твои конкретные юрлица и точки.

## Анонимность

Все значения в этом примере — плейсхолдеры (`ACME Corp`, `0000000000`,
`Example Street`). Никаких реальных данных. Твои реальные карточки будут
создаваться Архивариусом уже в твоём локальном `memory/`, который в
`.gitignore` и не попадёт в публичный репозиторий.
