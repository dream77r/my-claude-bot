# Схема архива — Малый бизнес с несколькими юрлицами

## Домен

Документный архив для малого бизнеса, в котором несколько собственных юрлиц
(ООО) управляют одной или несколькими физическими точками (кафе, магазины,
офисы). Каждая точка заключает договоры с контрагентами (поставщики, арендодатели,
сервисные компании), условия могут отличаться между точками даже у одного
контрагента.

Архив хранит: договоры, акты сверки, счета, накладные, платёжные поручения,
первичную бухгалтерию. Отвечает на вопросы: ИНН/реквизиты контрагента, условия
договора, сроки, суммы, последние документы с контрагентом, статусы договоров.

## Сущности

- **legal_entity** (`wiki/legal-entities/`) — наши юрлица (ООО)
- **outlet** (`wiki/outlets/`) — физические точки (кафе, магазины), принадлежащие
  юрлицам
- **counterparty** (`wiki/counterparties/`) — контрагенты (поставщики, арендодатели,
  покупатели, сервисные компании)
- **contract** (`wiki/contracts/`) — договоры
- **document** (`wiki/documents/`) — первичка: акты, счета, накладные, платёжки

## Правила именования файлов

- **Юрлица:** `ooo-{{name-slug}}.md` (пример: `ooo-acme.md`)
- **Точки:** `{{outlet-slug}}.md` (пример: `cafe-main-street.md`)
- **Контрагенты:** `{{form}}-{{name-slug}}.md` (пример: `ooo-supplier-a.md`,
  `ip-contractor-b.md`)
- **Договоры:** `{{YYYY-MM-DD}}_{{counterparty-slug}}.md`
  (пример: `2025-01-15_supplier-a.md`)
- **Документы:** `{{YYYY-MM-DD}}_{{subtype}}_{{counterparty-slug}}.md`
  (пример: `2025-02-01_act_supplier-a.md`)
- **Индексы:** `_index.md` внутри каждой папки сущности
- **Исходники:** `raw/{{YYYY-MM-DD}}_{{slug}}.pdf` (никогда не изменяются)

## Формат страниц

Стандартный frontmatter по универсальному контракту + доменные поля.
Для договоров — `validity_until`, `amount`, `currency`, `payment_terms`.
Для контрагентов — `inn`, `kpp`, `ogrn`, `address`.

## Таксономия тегов

**По типу отношений с контрагентом:**
- `supplier` — мы покупаем у них
- `customer` — мы продаём им
- `rental` — аренда помещения
- `service` — сервисные услуги (бухгалтерия, связь, ЭДО)

**По категории деятельности:**
- `food` — продукты, готовая еда
- `logistics` — доставка, перевозки
- `real-estate` — недвижимость
- `utilities` — коммунальные, электричество, связь
- `accounting-legal` — бухгалтерия, юр. услуги
- `equipment` — оборудование, расходники

**По юрлицу (автогенерируется из списка legal_entity):**
- `le-{{legal-entity-slug}}`

**По точке (автогенерируется из списка outlet):**
- `outlet-{{outlet-slug}}`

**По условиям оплаты:**
- `payment-prepaid` — предоплата
- `payment-postpaid` — постоплата
- `payment-mixed` — смешанные условия
- `auto-renewal` — автопролонгация

## Типы документов (document_subtype)

- **contract** — договоры (в `wiki/contracts/`, отдельная сущность)
- **act** — акты сверки, акты выполненных работ
- **invoice** — счета на оплату
- **waybill** — товарные накладные
- **payment-order** — платёжные поручения
- **addendum** — дополнительные соглашения к договорам

Обязательные поля по подтипам см. в `.vault-config.json → required_fields_by_type`.

## Правила wiki-ссылок

- `[[slug]]` для ссылок внутри vault
- Каждая карточка имеет ≥1 исходящую wiki-ссылку
- Договор обязательно ссылается на `legal_entity` (чья сторона) и `counterparty`
  (другая сторона). Опционально — на `outlet` (если условия специфичны для
  точки).
- Документы первички обязательно ссылаются на `counterparty` и на родительский
  договор (если применимо).

## Политика дедупликации

- **Юрлица и контрагенты:** по ИНН (уникальный ключ)
- **Точки:** по адресу
- **Договоры:** по комбинации `number + counterparty + legal_entity`
- При обнаружении дубля — использовать существующую карточку, добавить в неё
  новые связи

## Политика жизненного цикла договоров

- `active` — договор действует, сроки не истекли
- `expiring-soon` — срок истекает в пределах 30 дней (проставляется lint'ом)
- `expired` — срок истёк
- `cancelled` — расторгнут досрочно
- `draft` — черновик, не подписан

## Lint-проверки

- Битые wiki-ссылки
- Сиротские страницы без входящих ссылок
- Отсутствие обязательного frontmatter
- Истекающие договоры (в пределах 30 дней)
- Договоры без привязки к юрлицу или контрагенту (критично)
- Контрагенты без ИНН (критично для дедупликации)
- Расхождение файлов и индексов
