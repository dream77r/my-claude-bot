# Skill: Setup Team

## Когда активировать

Когда пользователь в Telegram просит настроить/установить/изменить агента `team` в групповом чате. Ты — master-агент `me`, ты проводишь диалог с пользователем и сам записываешь конфигурацию в память `team`.

## Принцип

Никакого хардкода. Все параметры берутся из диалога с пользователем. Результат — единый файл `../team/memory/groups/{chat_id}/settings.json` и опциональные entity-страницы для участников.

Пользователь может не знать технических деталей — задавай вопросы простым языком, на языке пользователя (проверь его предыдущие сообщения).

## Диалог онбординга — пошагово

### Шаг 1. Подтвердить контекст

Спроси (или подтверди из контекста):
- «В какой группе настраиваем team?» → нужен `chat_id` (числовой id чата в Telegram).
- «В каком топике (теме) он будет работать?» → `message_thread_id`. Если группа без топиков — `null`.
- «Ты — владелец этой группы?» → `owner_user_id`, `owner_username`.

**Как получить chat_id и thread_id технически:** обычно пользователь не знает их наизусть. Предложи ему переслать любое сообщение из нужной группы и топика в твой DM — из пересланного сообщения ты сможешь извлечь id. Или попроси добавить team в группу и упомянуть его там (тогда `team` сам сохранит chat_id в `pending_topic_setups`).

Если пользователь не готов — скажи: «Хорошо, настроим позже. Когда будешь готов — напиши “настрой team”».

### Шаг 2. Таймзона и язык

- «В какой таймзоне работает команда?» (например `Europe/Moscow`, `America/Los_Angeles`). Дефолт — `UTC`.
- «На каком языке команде удобнее — русский или английский?» Дефолт — язык пользователя.

### Шаг 3. Расписание синхронизаций

- «Нужны ли утренние и вечерние стендапы?» (yes/no)
- Если да:
  - «Во сколько утром?» (дефолт `09:00`)
  - «Во сколько вечером?» (дефолт `18:00`)
  - «Только по будням?» (yes/no, дефолт yes)
  - «Нужно ли ночное саммари?» (yes/no, дефолт yes, время 23:30)

### Шаг 4. Поведение в чате

- «Team должен отвечать только когда его упоминают или ещё и на общие вопросы?» → `respond_mode: "mention_only" | "mention_and_questions"`. Дефолт — `mention_only`.
- «Team может тегать участников в стендапах?» → `allow_tagging: true | false`. Дефолт — `true`.

### Шаг 5. Приватность

- «Могут ли участники видеть обещания и задачи друг друга?» → `privacy.hide_from_others: false | true`. Дефолт — `false` (всё видно всем).
- «Кто может менять настройки team в этой группе?» → `admins: [owner_username]` + другие. Дефолт — только владелец.

### Шаг 6. Участники (опционально)

«Перечисли участников команды — имя и tg username (например: Аня @anya, Боб @bob). Это не обязательно, team сам их добавит при первом сообщении каждого в группе».

Если пользователь даёт список — создай entity-страницы (см. ниже).

### Шаг 7. База знаний компании (опционально)

«Есть ли текстовый документ о компании/продукте/команде, который хочешь дать team для Q&A? Можешь прислать сейчас или потом через `/upload_knowledge`».

Если пришлют — сохрани в `../team/memory/wiki/concepts/company/<filename>.md`.

## Что создать в памяти

### 1. `../team/memory/groups/{chat_id}/settings.json`

```json
{
  "chat_id": <int>,
  "message_thread_id": <int_or_null>,
  "allowed_topic": <int_or_null>,
  "owner_user_id": <int>,
  "owner_username": "<string>",
  "admins": ["<username>"],
  "language": "ru",
  "timezone": "Europe/Moscow",
  "respond_mode": "mention_only",
  "allow_tagging": true,
  "daily_sync": {
    "enabled": true,
    "morning_time": "09:00",
    "evening_time": "18:00",
    "weekdays_only": true,
    "nightly_summary_time": "23:30",
    "day_end_time": "18:00",
    "overdue_followup_enabled": true,
    "morning_message_style": "friendly"
  },
  "privacy": {
    "hide_from_others": false
  },
  "onboarded_at": "<ISO timestamp>",
  "onboarded_by": "<username>"
}
```

Используй `Write`. Перед записью убедись, что папка `groups/{chat_id}/daily/` и `groups/{chat_id}/wiki/` существуют (создай через Bash `mkdir -p`).

### 2. Entity-страницы (если пользователь дал список)

Для каждого `Имя @username` создай `../team/memory/wiki/entities/<username>.md`:

```markdown
---
name: <Имя>
tg_username: <username>
tg_user_id: null
role: null
active: true
away_until: null
added_at: <ISO>
added_by: <owner_username>
---

# <Имя>

Член команды. Роль и зона ответственности будут уточнены при первом сообщении в группе.
```

### 3. Обновить `../team/memory/index.md`

Добавь ссылку на новую группу в секцию «Группы» (создай, если нет).

### 4. Подтверждение в чат

После записи всех файлов — дай пользователю короткое подтверждение:

```
✅ Team настроен в группе <название>.

Расписание:
- Утренний стендап: 09:00 (пн-пт)
- Вечерний дайджест: 18:00 (пн-пт)
- Ночное саммари: 23:30

Участников зарегистрировано: N
Топик: <thread_id или "вся группа">
Таймзона: Europe/Moscow

Чтобы изменить — напиши "настрой team" ещё раз.
```

## Изменение существующих настроек

Если `settings.json` уже существует:
1. Прочитай его.
2. Спроси, что именно менять (расписание / таймзону / топик / участников / приватность).
3. Обнови только нужные поля через `Edit` (не перезаписывай весь файл).
4. Обнови `onboarded_at` → `updated_at`.

## Ограничения

- Только master-агент `me` может запускать этот скилл. У `team` к нему доступа нет.
- Не создавай `settings.json` без `chat_id`. Если пользователь не смог его дать — останавливайся.
- Никогда не пиши личные данные (пароли, токены) в память team.
- Если `message_thread_id` не указан — team будет отвечать во всей группе, что может создавать шум. Предупреди пользователя.
- Если `morning_time == evening_time` — переспроси, это ошибка.
