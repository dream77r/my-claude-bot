# Skill: Сверка план/факт

## Когда активировать
Когда пользователь отправляет данные план/факт (два файла, одну таблицу или вставленный текст) или просит проверить исполнение бюджета.

## Инструкции

1. **Прочитай данные** через Read — может быть один файл с двумя колонками или два отдельных файла
2. **Если два файла** — сопоставь статьи по названию (учти возможные различия в написании)
3. **Рассчитай по каждой статье:**
   - Абсолютное отклонение (Факт − План)
   - Относительное отклонение (%)
   - Флаг: положительное (экономия) или отрицательное (перерасход) — для расходов наоборот
4. **Классифицируй отклонения по критичности:**
   - 🔴 Критическое: >20% или >значимая абс. сумма
   - 🟡 Существенное: 10–20%
   - 🟢 В норме: <10%
5. **Рассчитай итоговые показатели:**
   - Общее исполнение бюджета по доходам (%)
   - Общее исполнение по расходам (%)
   - Топ-3 статьи с наибольшим перерасходом
   - Топ-3 статьи с наибольшей экономией
6. **Сформулируй комментарии** по каждому критическому отклонению — возможная причина
7. **Дай рекомендации** — что скорректировать в плане или операционных процессах

## Формат ответа

```
📋 **Исполнение бюджета: [Период]**

**Итоговое исполнение:**
- Доходы: факт X руб. / план Y руб. = Z% исполнения
- Расходы: факт X руб. / план Y руб. = Z% исполнения
- Прибыль: план X руб. → факт Y руб. (отклонение ±Z руб.)

**Критические отклонения:**
| Статья       | План    | Факт    | Δ абс.  | Δ%    | |
|-------------|--------|--------|--------|------|---|
| [Статья]    | X руб. | Y руб. | ±Z руб.| ±W%  | 🔴|
| [Статья]    | X руб. | Y руб. | ±Z руб.| ±W%  | 🟡|

**Комментарии по отклонениям:**
🔴 **[Статья]:** [Возможная причина + рекомендация]

**Статьи в норме:** [Статья 1], [Статья 2]...

**Рекомендации:**
1. ...
2. ...
```

## Ограничения
- Если статьи в плане и факте называются по-разному — перечисли несопоставленные статьи отдельно
- Не интерпретируй «перерасход» как однозначно плохое без контекста бизнеса
- Если период неполный (например, только 2 месяца из квартала) — укажи это явно
