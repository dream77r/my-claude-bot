# Skill: Саммари движения денег

## Когда активировать
Когда пользователь отправляет банковскую выписку, ОДДС (отчёт о движении денежных средств) или просит разобраться с денежными потоками.

## Инструкции

1. **Прочитай данные** через Read (файл) или из текста сообщения
2. **Определи период** и начальный/конечный остаток
3. **Разбей потоки на три группы** (если данных достаточно):
   - Операционная деятельность: выручка от продаж, оплата поставщикам, зарплата, аренда
   - Инвестиционная: покупка/продажа активов, оборудования
   - Финансовая: кредиты, займы, дивиденды
4. **Рассчитай ключевые показатели:**
   - Суммарные поступления и списания за период
   - Чистый денежный поток (поступления − списания)
   - Топ-3 источника поступлений (по сумме)
   - Топ-3 статьи расходов (по сумме)
   - Среднедневной остаток на счёте
5. **Найди кассовые разрывы:**
   - Дни/периоды, когда остаток опускался ниже безопасного уровня
   - Задержки поступлений на фоне постоянных расходов
   - Пиковые дни расходов (зарплата, налоги, аренда)
6. **Выяви паттерны:** сезонность, регулярность крупных платежей, нерегулярные поступления
7. **Сформулируй рекомендации** по управлению ликвидностью

## Формат ответа

```
💰 **Движение денег: [Период]**

**Сводка:**
| Показатель              | Значение     |
|------------------------|-------------|
| Остаток на начало       | X руб.      |
| Поступления             | +X руб.     |
| Списания                | −X руб.     |
| Чистый поток            | ±X руб.     |
| Остаток на конец        | X руб.      |

**Топ поступлений:**
1. [Контрагент/Статья] — X руб. (Y%)
2. ...

**Топ расходов:**
1. [Контрагент/Статья] — X руб. (Y%)
2. ...

**Кассовые разрывы:**
⚠️ [Дата/период] — остаток X руб., причина: [...]
_(или: кассовых разрывов в периоде не обнаружено)_

**Паттерны:**
- [Наблюдение о регулярности, сезонности]

**Рекомендации:**
1. ...
2. ...
```

## Ограничения
- Если выписка содержит только суммы без назначений — укажи это и анализируй что есть
- Не классифицируй платёж без уверенности — лучше вынеси в «прочее»
- При наличии нескольких счетов — анализируй каждый отдельно, если не указана консолидация
