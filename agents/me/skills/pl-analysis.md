# Skill: Анализ P&L

## Когда активировать
Когда пользователь отправляет P&L-отчёт (CSV, Excel, текст, PDF) или просит проанализировать финансовые результаты компании.

## Инструкции

1. **Прочитай данные** через Read (файл) или из текста сообщения
2. **Определи период** — месяц, квартал, год; есть ли динамика (несколько периодов)
3. **Рассчитай ключевые метрики:**
   - Выручка и её динамика (% изм. к предыдущему периоду)
   - Валовая прибыль и маржа (Выручка − Себестоимость) / Выручка
   - EBITDA и EBITDA-маржа (если есть данные по амортизации)
   - Операционная прибыль / убыток
   - Чистая прибыль / убыток и рентабельность
   - Топ-3 статьи расходов по доле в выручке
4. **Найди аномалии:**
   - Резкие изменения статей (>20% месяц к месяцу без видимой причины)
   - Отрицательная маржа на отдельных продуктах/направлениях
   - Расходы, растущие быстрее выручки
   - Необычные разовые статьи
5. **Если есть несколько периодов** — покажи тренд: рост/падение, ускорение/замедление
6. **Сформулируй рекомендации** — 3-5 конкретных действий на основе данных
7. Если данные важны — сохрани краткое резюме в wiki/concepts/

## Формат ответа

```
📊 **P&L Анализ: [Компания / Период]**

**Ключевые метрики:**
| Показатель       | Значение     | Динамика |
|------------------|-------------|---------|
| Выручка          | X руб.      | +Y%     |
| Валовая маржа    | X%          | ±Y п.п. |
| EBITDA           | X руб.      | +Y%     |
| Чистая прибыль   | X руб.      | +Y%     |

**Структура расходов (топ-3):**
1. [Статья] — X% от выручки
2. [Статья] — X% от выручки
3. [Статья] — X% от выручка

**Аномалии / На что обратить внимание:**
⚠️ [описание аномалии и возможная причина]

**Тренд** (если несколько периодов):
[Краткое описание динамики]

**Рекомендации:**
1. ...
2. ...
3. ...
```

## Ограничения
- Если данных недостаточно для расчёта метрики — явно отметь «нет данных»
- Не делай отраслевых сравнений без дополнительных данных
- При неоднозначных статьях — уточни у пользователя, не додумывай
